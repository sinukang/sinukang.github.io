페어 프로그래밍을 위한 웹 IDE 및 관리 시스템 | 2025.01.06 ~ 2025.02.21

프로젝트 개요

페어 프로그래밍을 할 때 공간적 제약, 생산성, 비효율적인 관리 시스템, 테스트 환경을 해결하기 위한 웹 IDE 공동편집 및 관리 시스템입니다. 웹 IDE를 통한 실시간 공동 코드 편집, 화상 통화, 채팅 등의 기능을 제공하여 효율적인 페어 프로그래밍 환경을 구축합니다.

React
TypeScript

주요 기능

  • 웹 IDE 동시편집 - 실시간으로 코드를 함께 작성하고 수정할 수 있는 기능
  • 실시간 영상통화 및 채팅 - 원활한 의사소통을 위한 화상 및 텍스트 채팅 기능
  • 프로젝트 관리
    • 프로젝트 생성시 옵션설정(언어, 환경, 스펙 등등)
    • 프로젝트 on/off
    • 프로젝트 사용량 체크
  • AI 코드 코칭 - ChatGPT API를 활용한 코드 개선 및 학습 지원

기술 스택 선정 이유

  • 리액트 : 웹 IDE를 하는 만큼 레퍼런스와 라이브러리가 많이 필요하기 때문에 선택
  • 타입스크립트 : 팀이 FE담당이 많아 디버깅을 제대로 해야 할 필요가 있어서 타입스크립트로 선정
  • 테일윈드 CSS : 빠른 CSS 적용으로 개발기간 단축
  • OpenVidu : 실시간 화상 채팅에 사용
  • SockJS, StompJS : 스프링 부트와 웹소켓 통신을 자주하기 때문에 사용

담당 역할

  • 프로젝트 관리(FE) - 프로젝트 생성, 옵션 설정, 상태 관리 기능 개발
  • 채팅 기능 - SockJS와 StompJS를 활용한 실시간 채팅 기능 구현
  • AI 코드 코칭 - ChatGPT API를 활용한 코드 분석 및 개선 제안 기능 개발

프로젝트 성과

  • ChatGPT API를 활용한 코드 코칭 기능 구현
  • Jira 협업툴을 통한 체계적인 프로젝트 관리 경험
  • 리액트 및 타입스크립트 실무 활용 능력 향상
  • FE팀원이 많은 환경에서 Git을 활용한 형상관리 및 협업 방식 개선

프로젝트 리뷰 및 회고

리액트와 타입스크립트를 처음 접해본 프로젝트였습니다. 처음에는 핵심기능이나 복잡한 기능을 담당하는 것에 두려움이 있었지만, 지금 돌아보면 더 도전적인 기능을 맡았으면 더 많이 성장할 수 있었을 것이라는 아쉬움이 있습니다.

프로젝트 초반에는 협업 관련 트러블이 있었으나, 중반부터는 팀원들이 모두 협업 관리를 효율적으로 하게 되면서 이상적인 협업 형태를 경험할 수 있었습니다. 특히 이번 프로젝트에서는 AI를 활용한 코드 작성 및 레퍼런스 활용이 많아지면서 새로운 트러블이 발생했고, 이를 해결하기 위해 AI 사용에 대한 컨벤션을 추가하는 등 개발 과정에서 AI를 효율적으로 활용하는 방법을 배울 수 있었습니다.
