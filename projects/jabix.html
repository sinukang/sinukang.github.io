다중 음성디바이스를 이용한 AI음성비서 시스템 | 2025.02.24 ~ 2025.04.11

프로젝트 개요

다중 디바이스로 실내 어디서든 음성비서를 이용하고, 집안에서 발생하는 소음을 분석하여 위험상황을 감지하는 시스템입니다. Wake up keyword를 통한 음성인식, SED 기술을 활용한 생활 소음 분류, 중앙 AP를 통한 클라우드 통신, IoT 데이터 시각화 및 제어가 가능한 PWA 어플리케이션을 제공합니다.

React
Tailwind CSS
SockJS
StompJS
PWA

구현 사항

  • Wake up keyword를 통해 음성인식
  • SED 기술로 생활 소음으로부터 특성 소리인지 분류하는 기능
  • 중앙 AP를 통해 해당 내용들을 종합하여 클라우드와 통신
  • 클라우드에서 가지고있는 IoT 데이터를 시각화하고 제어할 수 있는 어플리케이션(PWA)

기술 스택 선정 이유

  • 리액트 : PWA로 빠르게 개발하기 위해서 프론트 라이브러리로 사용
  • 테일윈드 CSS : 빠른 CSS 적용으로 개발기간 단축
  • SockJS, StompJS : 스프링 부트와 웹소켓 통신을 자주하기 때문에 사용
  • PWA : 실제 서비스보다 시각화가 우선이기 때문에 웹/앱 화면 둘다 필요해서 PWA로 구현

담당 역할

  • PWA 어플리케이션 개발(FE) - IoT 데이터 시각화 및 제어 기능 구현

프로젝트 성과

  • 리액트의 레퍼런스들을 많이 사용해봤음
  • Cursor AI를 사용했는데, 모르는 레퍼런스부분에 대해 코드를 짜는데 큰 도움을 많이 받았음
  • 백엔드와 IoT까지 통신에서 문제가 많이 발생했지만 끝내 해결할 수 있었음

프로젝트 리뷰 및 회고

하지만 프로젝트 막바지에 가서 내가 정한 로직내에서 짜지 않은 코드들에 코드가 더해지니 버그가 많이 발생했습니다. AI 코드를 짤 때 해당 영역을 정해야 한다는 것을 배웠습니다.

처음에 이걸 연구적인 성격의 프로젝트로 잡을지 서비스기반의 프로젝트로 잡을지 정하는 과정에서 시간을 많이 사용했습니다. 이 시간을 아낄 수 있었다면 더 좋은 성과를 낼 수 있었을 것 같습니다.
